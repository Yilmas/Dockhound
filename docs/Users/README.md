# Dockhound — User Guide

**Top-level invocation prefix:** `/dock`

This section documents commands and interactions available to regular server members. It is split into two categories:

- **Whiteboard** — collaborative whiteboards and their management.
- **Verification** — the verification flow for users requesting recruitment/ally status.

Navigate into each category for detailed guides and examples.

## Contents

### Whiteboard
- Overview & examples: `docs/Users/Whiteboard/README.md`
- Commands: `/dock whiteboard create`, `/dock whiteboard mode`, `/dock whiteboard roles`, `/dock whiteboard info`, `/dock whiteboard archive`, `/dock whiteboard list`

### Verification
- Overview & flow: `docs/Users/Verification/README.md`
- Context commands: Request Recruit, Request Ally (see subpages in the Verification folder)
